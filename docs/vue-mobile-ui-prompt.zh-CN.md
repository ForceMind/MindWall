# MindWall 前端重构提示词规范（Vue 移动端）

> 用于约束用户端与后台端 UI/交互实现，避免页面堆叠、样式随意、文案混杂。

## 1. 总体目标
- 技术栈固定为 Vue 3 + Vite + Pinia + Vue Router。
- 语言固定为简体中文，不向用户暴露后端英文报错。
- 体验固定为移动端优先，同时兼容 PC。
- 页面固定为“单任务页面流”，禁止在一个页面堆叠多个流程步骤。

## 2. 用户端规范
- 未登录用户只能看到：登录页、注册页。
- 登录后按状态跳转：
  - `onboarding`：资料 -> 访谈 -> 城市（分三页）
  - `active`：进入匹配列表
  - `restricted`：进入受限页
- 主业务结构：
  - 匹配列表页（会话列表 + 潜在匹配）
  - 会话详情页（单会话）
- 不使用“聊天/联系人”双 Tab；以“列表进入详情”作为唯一主路径。
- 会话中不显示“AI 用户/真实用户”身份标签。

## 3. 后台端规范
- 必须有管理员登录页。
- 登录后按二级分类路由组织：
  - 总览
  - 用户
  - 在线
  - AI记录
  - 提示词
  - 系统配置
  - 服务器日志
- 配置页必须可直接填写：
  - API Base URL
  - API Key
  - 聊天模型
  - Embedding 模型
  - Web Origin
- 配置页必须有“连通性测试”并展示结果细项。

## 4. 视觉规范
- 避免绿色主色，使用深色底 + 高对比强调色。
- 选中态文字必须为白色，禁止深色底深色字。
- 卡片、按钮、状态徽章需统一圆角、边框、间距体系。
- 不使用“手机外壳模拟器”容器，采用原生响应式布局。

## 5. 文案与错误处理
- 后端错误统一映射为中文可读提示。
- 禁止直接展示 `400/401/500` 及后端原始报错结构给终端用户。
- 管理后台可展示技术细节，但默认文案仍为中文。

## 6. 交互规则
- 每个页面只承担一个主要任务。
- 关键操作必须有明确反馈：加载中、成功、失败。
- 长列表必须支持刷新和分页/增量展示。
- 聊天页必须支持回车发送、滚动到底、系统消息区分。

## 7. 工程规则
- 前端环境变量统一使用 `VITE_*` 前缀。
- 运行入口为 `scripts/start-local.cmd`。
- 部署更新入口为 `scripts/deploy-update.cmd`。
- 前端构建通过 `npm run build` 作为交付前硬校验。
